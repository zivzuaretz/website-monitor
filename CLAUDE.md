# CLAUDE.md — הקשר מלא לפרויקט

## מה הפרויקט הזה

שני שכבות נפרדות שחיות באותו ריפו:

1. **מוניטור אתרים** (`monitor.py`) — סקריפט פייתון שרץ פעמיים ביום דרך GitHub Actions ובודק שינויים בדפי אתרים של חברות ביטוח והשקעות. כשיש שינוי — שולח מייל. זה מנגנון רקע, לא נוגעים בו בלי סיבה.

2. **דף מעקב תחרות** (`index.html` + `data.json`) — דף נחיתה ציבורי שמוצג דרך GitHub Pages. הדף מציג לוח דירוג חי של תחרות מכירות פנימית בחברת אלטשולר שחם. **זה הדבר הפעיל שמתוחזק שוטף.**

## איך מתחזקים את הדף

### קובץ הנתונים: `data.json`
**זה הקובץ היחיד שנוגעים בו לעדכון נתונים שוטף.**

```json
{
  "refresh_date": "YYYY-MM-DDTHH:MM:SS+03:00",
  "phase": 1,
  "group_target": 80,
  "team_managers": [...],
  "supervisors": [
    { "name": "שם", "team": "שם מנהל", "points": 0, "x2_count": 0, "league": null }
  ]
}
```

**שדות:**
- `refresh_date` — **חובה** לעדכן לזמן ישראל המדויק בכל עדכון. קבל עם: `TZ='Asia/Jerusalem' date +"%Y-%m-%dT%H:%M:%S+03:00"`
- `phase` — `1` או `2`
- `points` — סה״כ נקודות (כולל X2 כבר מחושב)
- `x2_count` — כמות נקודות בונוס בלבד מ-X2 (לתצוגה בעמודה)
- `league` — `null` בשלב א׳; בשלב ב׳: `"hungry"` או `"warming"`

### תהליך עדכון (בכל עדכון נתונים)
```bash
# 1. קבל שעה מדויקת
TZ='Asia/Jerusalem' date +"%Y-%m-%dT%H:%M:%S+03:00"

# 2. עדכן data.json (שמות, נקודות, refresh_date)

# 3. commit ו-push
git add data.json
git commit -m "data: עדכון נקודות DD/MM"
git push origin main
```

### מעבר לשלב ב׳ (סוף מאי)
1. עדכן `"phase": 2` ב-`data.json`
2. עדכן `"league": "hungry"` או `"league": "warming"` לכל מפקח לפי הדירוג הסופי של שלב א׳
3. הדף יציג אוטומטית את הליגות הנכונות

## כללי עבודה חשובים

- **עדכון נתונים = עדכון `data.json` בלבד** + push ל-main
- **תמיד** לעדכן `refresh_date` לשניה המדויקת בשעון ישראל
- **לא** לגעת ב-`monitor.py`, `urls.txt`, `snapshots.json` ללא בקשה מפורשת
- **לא** לדחוף ל-branch שאינו main (GitHub Pages מגיש מ-main)
- כשהמשתמשת מוסרת נתונים (נקודות, X2), לעדכן `data.json` ולדחוף מיד
- הדף נבנה ע״י Claude Code בשיחה פעילה — המשתמשת היא זיו זוארץ (מנהלת הצוות בתחרות)
